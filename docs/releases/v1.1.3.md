# Tunnara Platform 1.1.3

A versão `1.1.3` corrige o versionamento da pipeline após os hotfixes da série `1.1.2`.

## Principais correções

- versionamento automático após merge em `main`;
- incremento `patch` por padrão, com labels para `major`, `minor`, `patch` e ausência de release;
- sincronização dos pacotes Node, crates Rust, Tauri, Control API, SDK C, Android, iOS e imagens Docker antes do build;
- releases publicadas e tags tratadas como imutáveis;
- retomada exclusiva de drafts da mesma versão por `release_id`;
- bloqueio de drafts duplicados e de tentativa de republicar uma versão já publicada;
- correção do erro `HTTP 404` observado após reabrir `v1.1.2` como draft.

## Fluxo de release

Após um Pull Request ser mesclado em `main`, o workflow `Version and release after merge` calcula a próxima versão, executa `npm run version:set`, grava a versão sincronizada na branch principal e dispara o workflow coordenado `Release after merge`. A publicação só ocorre depois que Core, Runtime, SDK, Desktop, Mobile e Containers terminarem com sucesso.
