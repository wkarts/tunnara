# Tunnara Platform 1.1.2

Esta versão corrige as incompatibilidades de dependências identificadas no release pós-merge da série 1.1.1 e endurece o CI para impedir que Pull Requests de dependências sejam aprovados sem compilar os componentes afetados.

## Principais correções

- Runtime Rust e QUIC Bridge voltam a resolver `reqwest 0.13` com a feature TLS correta e toolchain Rust 1.85 compatível.
- Console Tauri volta a compilar em Linux, Windows e macOS com versões compatíveis de `rand`, `sha1`, `sha2` e `hmac`.
- Android usa a combinação AGP 9.2.1 / Gradle 9.4.1.
- iOS usa checkout local fixado do WireGuardKit, manifesto Swift Package corrigido, Go 1.19 e target legado para construir `wireguard-go` no SDK/arquitetura corretos.
- Workflows de Pull Request passam a compilar Rust, Android e o aplicativo iOS Simulator, além de resolver o pacote WireGuardKit.
- GitHub Actions Docker/Release foram atualizadas para versões com runtime Node 24.
- Dependabot não abre mais uma tempestade de PRs major independentes; atualizações minor/patch são agrupadas e limitadas.
- Correções posteriores do próprio pipeline não são mais encerradas silenciosamente com todos os jobs em `skipped`: o push do hotfix reabre a release atual em draft, atualiza a tag e força a reconstrução coordenada.

## Operação

O merge desta versão em `main` altera `VERSION` para `1.1.2`, disparando o pipeline coordenado de release. Correções posteriores nos workflows e scripts de empacotamento também disparam uma reconstrução automática da versão corrente, mesmo quando a release já estiver publicada. A release permanece em draft até Runtime, SDK, Desktop, Mobile e Containers terminarem com sucesso.
