# Tunnara 2.0.0-rc.3

## Objetivo

Corrigir a falha do build pós-merge que interrompia o job de Core antes da geração dos executáveis SEA e dos demais artefatos da release.

## Causa raiz

O script `scripts/release/build-sea.mjs` executava:

```text
node node_modules/esbuild/bin/esbuild ...
```

O arquivo `node_modules/esbuild/bin/esbuild` é um executável nativo ELF no Linux e PE no Windows, não um módulo JavaScript. O Node tentou interpretar o cabeçalho binário como código e encerrou com:

```text
SyntaxError: Invalid or unexpected token
```

## Correção

- Uso direto da API JavaScript `build()` exportada pelo pacote `esbuild`.
- Bundles CJS do Agent e Server gerados de forma portável.
- `postject` continua sendo executado pelo seu CLI JavaScript.
- Mensagens de erro aprimoradas para falha de spawn e status ausente.
- Novo preflight `validate:sea` executado no CI de Pull Request sem gerar artefatos persistentes.
- Validador da release impede o retorno da chamada incorreta ao binário nativo.

## Versionamento

A versão foi elevada para `2.0.0-rc.3`. A release e a tag `v2.0.0-rc.2` permanecem imutáveis.

## Resultado esperado após o merge

1. O workflow de versionamento preserva `2.0.0-rc.3` quando a maior versão publicada for `2.0.0-rc.2`.
2. Uma única draft `v2.0.0-rc.3` é criada.
3. Core gera Console, SDK C, Agent e Server SEA.
4. Runtime, SDK, Desktop, Mobile e Containers são iniciados somente após o sucesso do Core.
5. A release é publicada após todos os grupos obrigatórios concluírem com sucesso.
