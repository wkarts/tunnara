# Tunnara 2.0.0-rc.2

## Objetivo

Corrigir integralmente as validações observadas nos logs, estabilizar builds multiplataforma e concluir o autoincremento de versão sem reutilizar releases ou tags publicadas.

## Correções principais

- `esbuild` explícito no Console e lockfile sincronizado.
- SemVer/prerelease consistente em todos os 25 pontos versionados.
- Próxima versão calculada a partir de releases, drafts e tags existentes.
- Releases publicadas e tags imutáveis; drafts retomáveis somente no mesmo SHA.
- Android com Kotlin integrado do AGP 9.2.1 e dependências compatíveis com API 35.
- iOS com versão numérica, build monotônico, `Info.plist`, WireGuardKit/Xcode 16 e simulador arm64.
- SEA Windows sem wrappers `.cmd`.
- Uploads sequenciais, idempotentes, portáveis e sem Actions Artifact Storage.
- Uma única draft compartilhada pelo Core, Runtime, SDK, Tauri e mobile.
- Assets Android/iOS/Runtime/SDK com nomes exclusivos.
- Containers RC não sobrescrevem `latest`.
- Dependabot agrupado e limitado a atualizações compatíveis.

## Versionamento automático

Sem label:

- RC atual: `2.0.0-rc.2` → `2.0.0-rc.3`;
- versão estável: incrementa o patch.

Labels:

- `release:prerelease`
- `release:stable`
- `release:patch`
- `release:minor`
- `release:major`
- `release:none`

## Classificação

Release Candidate para homologação e produção controlada. Testes em domínio real, dispositivos físicos, infraestrutura multi-host, pentest e soak test continuam sendo gates externos para GA.
