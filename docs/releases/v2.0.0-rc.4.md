# Tunnara 2.0.0-rc.4

## Objetivo

Corrigir a validação pós-merge causada por um overlay Docker legado mantido na
cópia local do projeto e tornar esse overlay parte oficialmente suportada do
perfil distribuído com QUIC.

## Causa raiz

O arquivo `deploy/docker/docker-compose.distributed.quic.yml` não existia no
pacote oficial RC.3, mas estava presente na versão local enviada ao GitHub. O
arquivo ainda continha a imagem:

```text
ghcr.io/wkarts/tunnara-quic-bridge:2.0.0-rc.2
```

Como `scripts/version/check-version.mjs` verifica todas as imagens versionadas
em `deploy/docker`, a CI interrompeu corretamente com:

```text
deploy/docker/docker-compose.distributed.quic.yml: imagem 2.0.0-rc.2 != 2.0.0-rc.3
```

## Correção

- overlay atualizado e integrado ao `tunnara.sh`;
- composição combinada validada no Pull Request;
- perfil distribuído padrão usa Relay TCP;
- perfil distribuído QUIC ativa o bridge e a exportação do certificado Caddy;
- backups antigos `.bak` removidos;
- detecção de arquivos legados e Compose órfão adicionada ao validador;
- versão elevada para `2.0.0-rc.4`.

## Comandos

```bash
cd deploy/docker
./tunnara.sh init
./tunnara.sh preflight-distributed-quic
./tunnara.sh up-distributed-quic
./tunnara.sh status-distributed-quic
```

Backup e atualização:

```bash
./tunnara.sh backup-distributed
./tunnara.sh update-distributed-quic
```

## Classificação

Release Candidate destinada a homologação e produção controlada. Os gates de
maturidade descritos em `docs/security/MATURITY_GATES.md` continuam aplicáveis.
